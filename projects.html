UWaterloo Mars Rover Team

Student Design Team
Description

Role: Team Lead (formerly Embedded Software Developer)

The UW Mars Rover named Arthrobot is created in hopes to compete in the University Rover Challenge (held in Hanksville, Utah) and European Rover Challenge (held in Poland). The Arthobot is a four-wheel independent steering exploration robot that can travers through rough terrain, analyze soil samples for signs of life, assist astronauts in various tasks, and manipulate equipment on-site equipment using its fine motion manipulator. Development of the rover is done on the ROS (Robotics Operating System) Open Source framework. A demonstration of the robots capabilities can be seen here as narrated by me.

Contributions
  • Research and developed four-wheel independent steering system and swerve drive algorithm
  • Developed on-board multiaxis camera control system
  • Designed and created Arduino shield-like PCB to simply wiring for servos, sensors and allow for power extention of Orion RobotClaw Motor Controllers
  • Implemented safety features and bumper sensors to prevent harmful overextention of the rover's arm
  • Developed display to show crucial data while performing field tests
  • Implemented smart power distribution system allow safe power usage of the batteries on the rover

adVentuRe: Mutliaxis Robotic Telepresence System

Hack Completed: 50 Hours
Description

Project Website

adVentuRe was created by me and group of 3 other robot enthusiasts during the 2015 Hack The North event. The adventure VR is a telepresence system that allows for a user to immerse himself into a far off location through telepresence. The system consists of a Oculus Rift VR headset which is used in the base station where the user resides. At a far off distant location, a stereo camera is mounted on a multi axis gimble (which in our case was attached to a mobile robot). The head motions are recorded in the Oculus Rift's IMU and tranferred as motion on the camera gimble. The stereo camera relay data to the Oculus allowing for 3D immersion. My role in the team was the develop the software which allowed the camera's data streams two be displayed on the two different lenses of the Oculus headset. In addition, I also created software which read the IMU data and moved the gimble. A demonstration of the hack can be seen here

Materials
  • Oculus Rift
  • Ubuntu Latop with ROS (Robotics Operating System)
  • Gimble
  • 2 Webcam Camera
  • Mobile Robot (optional)

This hack landed me my first win at an Official Major League Hacking Event! We built a home security system using Microsoft's Azure and Kinect platforms and the Twitter API. Mini Collaborative Desktop Arm

Personal Project In Progress
Description

This is a small robotic arm you can use to help you sort out your messy desk! The robot has three different modes: Operator Mode, Teach and Autonomous Mode. Operator mode allows the user to control the robot's joint using a small controller board. Teach mode records the user's movements on the controller and autonomous mode repeats what it is tauch in a repetitive motions. Future developments will allow the robot to use vision algorithm to sort individual items (ex. pens, eraser, coins etc.) and will be using a vacuum pump instead of arm.

Materials
  • Arduino Mini Pro 5V
  • 5 x 10k Potentiometers
  • 5V Power Source
  • Wires, Button, Switch
  • Plastic chassis
  • LCD Monitor for testing

DogeBot: Exploration Quadruped

Personal Project In Progress
Description

4 legged quadruped platform for the reseach and development of autonomous robotic locomotion and artificial intelligence. The quadruped is outfitted with 12 standard analog servos that allow for the natural mammal movement of the Coxa, Femur and Tibia. Using an Inverse Kinematics algoritm (IK) and a closed loop feeback system, the quadruped is able to traverse irregular terrain. The system will use computer vision to detect alterations in the enviroment and learn from its mistakes in order to traverse terrain in a progressively efficient and safe manner.

Materials
  • Arduino UNO
  • Rasberry Pi 2 B
  • Logitech Webcam
  • 12 Analog Servos
  • Acryclic Glass chassis
  • 4 Metal Rods
  • 4 Rubber covers
  • 5V Power Source
  • Breakout Inertial Measurement Unit (IMU)
This intensive project was created in hopes to push the limits of my knowledge in robotic controls, computer vision and machine learning. Currently in the design phase.

Relevant School Project Completed
Description

Me and another classmate created a line following robot from scratch that was able to build a line following robot that uses a Hall Effect Sensor to detect the presence of a magnetic field.
